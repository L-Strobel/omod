class Omod(areaFile: File, osmFile: File, routingMode: RoutingMode = RoutingMode.BEELINE, cache: Boolean = true, cacheDir: Path = Paths.get("omod_cache/"), odFile: File? = null, gridPrecision: Double = 200.0, seed: Long? = null, bufferRadius: Double = 0.0, censusFile: File? = null, populateBufferArea: Boolean = true, val distanceCacheSize: Long = 400e6.toLong(), populationFile: File? = null, nWorker: Int? = null, gtfsFile: File? = null, carOwnershipOption: CarOwnershipOption = CarOwnershipOption.FIX)

Open-Street-Maps MObility Demand generator (OMOD) Creates daily activity schedules in the form of activity chains and dwell times.

Parameters

areaFile

GeoJSON of the focus area

osmFile

osm.pbf file that covers at least the focus area and buffer area

routingMode

Method of distance calculation used in the simulation

cache

Option to cache the routing matrix for subsequent runs

cacheDir

Directory where the cache should be located

odFile

Origin-Destination Matrix in GeoJSON format. Optional input used for calibration.

gridPrecision

Precision of the routing grid

seed

Random number seed

bufferRadius

Distance with which the focus area is buffered to obtain the buffer area

censusFile

GeoJSON containing census information (Population distribution in the model area)

populateBufferArea

Option to populate the buffer area with agents

distanceCacheSize

Maximum size of the routing matrix cache

populationFile

File that defines the distribution of socio-demographic features for the agent population

nWorker

Number of parallel coroutines that can be executed at the same time. NULL = Number of CPU-Cores available.

gtfsFile

GTFS location (Directory or .zip file)
